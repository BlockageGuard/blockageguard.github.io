⚠️ PoC: CORS Misconfiguration — alts.zone Admin (F-059)

Finding: F-059 · Severity: HIGH
Root cause: Access-Control-Allow-Origin: * combined with Access-Control-Allow-Credentials: true on /admin/session/signin
Impact: Any origin can read the response from the admin login endpoint, including session tokens or error data. Combined with CSRF (F-091), a malicious page can both submit credentials AND read the response cross-origin.
Click to run...

The broken header combination

Access-Control-Allow-Origin: *
Access-Control-Allow-Credentials: true

// Per CORS spec: credentials + wildcard origin is INVALID.
// Browsers may still allow it — a specific origin like
// Access-Control-Allow-Origin: https://alts.zone should be set.

Fix

// nginx.conf — restrict to own origin only:
add_header Access-Control-Allow-Origin "https://alts.zone" always;
add_header Access-Control-Allow-Credentials "true" always;
// Never use * with credentials:true
